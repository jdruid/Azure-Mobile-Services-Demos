Azure Mobile Services : Easy API's Working Example

Azure Mobile Services Mobile App

This demo sites shows you how to configure Mobile Services, add a custom api and use authentication via 3rd party services.

Create Mobile Service

  1. Login to the Azure Portal. Click on NEW -> MOBILE APP
  2. Enter a app service name for your clients to access the service.
  3. Done

Create Your Data Table(s)

  1. Click on the EASY TABLES.
  2. CONNECT A DATABASE - Follow steps to select existing or create new one.
  3. Agree to terms and initialize the app.
  4. Done

Modify Your Schema

  1. Click on the Easy Tables from the property blade.
  2. From there click NEW TABLE and enter the table information.
  3. You can modify the schema by clicking on the table name and clicking MODIFY SCHEMA.
  4. You can connect to the Database you just created by using a SQL client tool and your connection string.
  5. To find your connection string click on Database Settings in the property blade, you will see connection string.
  6. Windows: can use SQL Management Tools or Cloud Explorer (within Visual Studio), or Mac: SQLPro for MSSQL or Mac SQL Studio
  7. FYI: if you are connecting to the SQL server you will have to add your ip address to the server firewall.
  8. Done

Access Your Table API

  1. Click on the table and then click on EDIT SCRIPT icon in the navigation tab
  2. Visual Studio Online will open with access to your table api script.
  3. You can now create individual access routines (read, delete, update and insert...see GitHub for code examples)
  4. FYI: if you name your object and properties in your code, then no need to change anything in the api unless you need specific business logic
  5. Your endpoint will be: https://MOBILE SERVICE NAME.azurewebsites.net/tables/TABLE NAME
  6. Done

Create A Custom API

  1. From our Mobile Services black, click on the EASY API.
  2. Provide your custom API with a name and click CREATE
  3. Here you can build custom logic to query multiple tables or perform some other routine.
  4. Your endpoint will be: https://MOBILE SERVICE NAME.azurewebsites.net/api/API NAME
  5. Done

View Errors and Logs

  1. From our Mobile Services blade, click on the EASY TABLE or EASY API and choose STREAMING LOGS.
  2. All logs will be there.
  3. Done
